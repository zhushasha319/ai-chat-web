# 千问 API 配置指南

## 🚀 快速开始

### 1. 获取千问 API Key

1. **访问阿里云控制台**：https://dashscope.console.aliyun.com/
2. **开通服务**：
   - 登录阿里云账号
   - 开通"灵积模型服务 DashScope"
   - 选择合适的套餐（有免费额度）

3. **创建 API Key**：
   - 进入 API Key 管理页面
   - 点击"创建新的 API Key"
   - 复制生成的 API Key

### 2. 配置 API Key

编辑项目根目录的 `.env` 文件：

```bash
# 将下面的 your-qwen-api-key-here 替换为你的真实 API Key
QWEN_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 3. 启动服务

```bash
# 启动后端服务器
node server.js

# 或者使用批处理文件（Windows）
start-qwen-server.bat

# 启动前端（另一个终端）
npm run dev
```

## 🎯 支持的模型

- `qwen-turbo` - 快速响应，适合日常对话
- `qwen-plus` - 更强能力，适合复杂任务
- `qwen-max` - 最强模型，适合专业场景
- `qwen-math-plus` - 数学专用模型
- `qwen-coder-plus` - 代码专用模型

## 🔧 故障排除

### 常见错误

1. **API Key 错误**：
   ```
   错误信息：请设置千问 API Key
   解决方案：检查 .env 文件中的 QWEN_API_KEY 设置
   ```

2. **网络连接错误**：
   ```
   错误信息：fetch failed
   解决方案：检查网络连接，确认能访问阿里云 API
   ```

3. **权限错误**：
   ```
   错误信息：401 Unauthorized
   解决方案：检查 API Key 是否正确，是否已开通服务
   ```

### 调试技巧

- 查看服务器控制台日志了解详细错误信息
- 使用浏览器开发者工具查看网络请求
- 确认 .env 文件在项目根目录且格式正确

## 🔒 安全提示

- ⚠️ **不要将 API Key 提交到版本控制系统**
- ⚠️ **不要在前端代码中暴露 API Key**
- ⚠️ **定期更换 API Key**
- ⚠️ **设置合理的使用限制**

## 📝 自定义配置

你可以在 `server.js` 中修改：

```javascript
// 修改默认模型
const { messages, model = 'qwen-plus' } = req.body

// 添加系统提示
const qwenPayload = {
  model: model,
  input: {
    messages: [
      {
        role: 'system',
        content: '你是一个专业的AI助手，请用中文回答问题。'
      },
      ...messages
    ]
  },
  parameters: {
    result_format: 'message',
    incremental_output: true,
    max_tokens: 2000, // 限制输出长度
    temperature: 0.7  // 控制创造性
  }
}
```
